指令与开发规范 (Astell Guide)

项目开发全流程规范与内置核心程序功能介绍

系统核心程序功能说明

retrieve_astell_knowledge 智能知识检索

从 MemPalace 数据库中高精度检索项目记忆 (wing_Project_Memory)、专项攻坚经验 (wing_Special_Operation_Memory) 及官方 SDK。

参数: query (str), current_project (str, 可选), strict_mode (bool, 默认False)

record_astell_trace [强制] 原子化开发留痕

警告：AI 严禁口头汇报进度而不留痕。 凡涉及实现新逻辑、遇到报错或检索空白，必须立即调用此工具。 系统会根据 title 参数自动在 开发日志/ 或 问题记录/ 文件夹下创建原子 MD 文件。

参数: type (str: dev/problem/search), content (str), title (功能/模块标识符)

set_active_project [必须调用] 开工契约

所有工作的起点。 AI 必须在接收任务后立即调用，初始化项目环境。系统会自动校验 resource 与 behavior 目录，并建立原子化记录仓库。

参数: path (str), prefix (项目前缀, 如 NightVision)

suggest_deep_research 深度检索建议

当常规搜索无果时，自动生成针对 `Library/` 源码和专项分区的深度检索路径建议，引导主代理通过高价值路径获取解法。

参数: query (str)

solidify_memory 记忆固化申请

发起记忆固化申请。AI 无权直接写库，此工具仅提醒人类前往控制塔“记忆审批”进行人工审查后手动入库，确保长期记忆的纯净度。

参数: 无

register_resource_id 资源 ID 注册

将新定义的实体/物品/方块 ID 写入“信息记录”，自带重复校验机制，从源头杜绝资源 ID 冲突导致的注册覆盖。

参数: resource_id (str), description (str), file_path (str, 可选)

audit_for_solidification 固化前安全审计

在发起固化前自动对比“开发日志”与“问题记录”，标记尚未解决的报错并提取“搜索日志”中的知识空白，防止脏数据或带 Bug 的方案误入库。

参数: 无

list_knowledge_inventory 知识清单概览

列出记忆宫殿中所有知识分区 (Wings) 及其覆盖范围。建议在盲目搜索前先调用，了解知识库边界以制定更精准的检索策略。

参数: 无

update_project_prefix 项目前缀更新

当 AI 对项目有更深理解后，重命名当前的记录文件前缀，并自动同步重命名现有的四件套文档，保持命名一致性。

参数: new_prefix (str)

open_control_tower 控制塔启动

在浏览器中打开 Astell 可视化控制塔，供人类进行记忆审批、查看知识库向量化状态及管理项目。

参数: 无

Astell 核心标准开发与固化规章

1

项目初始化与四件套锁定

根据项目指定并对接四个关键文档：

  • 开发日志：记录开发对应模块时使用的关键方法与时间。
  • 信息记录：记录注册的资源 ID 避免重复冲突。
  • 问题记录：仅记录严重报错和反馈问题。
  • 搜索日志：只在检索不到对应内容时记录，用于辅助后期固化决策。
2

分辨任务类别

对派发的开发需求进行分类：“新增内容”（如设计新实体等，必须同步到信息记录中）、“优化功能”或“反馈之前的报错”（只有在此状态下才动用问题记录）。

3

开发与按需调用

调用网易 Py 脚本或 API 前，务必先在知识库中检索确认；如果一个思路尝试很多次无果，请果断记录在搜索日志里。

4

计入问题记录

在遭遇 Bug 时，于问题记录中详细登记：错误现象、每一次尝试的方法与结果、以及最终解决问题的核心手段。

5

原子化原子日志 (Atomic Logging)

严禁将所有日志塞进一个文件。必须为每一个功能或 Bug 命名（如：夜视贴图渲染），并确保在 开发日志/ 文件夹下生成同名 MD 文件。

6

文件名级交叉审计

系统会自动将 开发日志/Module.md 与 问题记录/Module.md 进行对齐。若该功能仍存在未闭环报错，固化流程将被强制拦截。

7

人工状态标记

人类可在“工作区记录”视图中手动标记某个记录为“不可信”。被标记的记录将永久失去入库资格，防止错误经验污染主脑。

8

人工最终审批 (Human Audit)

所有的物理入库动作必须通过“记忆审批”完成。在这里，您可以过滤废话、提纯逻辑，确保只有高质量经验才进入长期知识库。
