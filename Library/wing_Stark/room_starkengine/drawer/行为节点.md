# [StarkEngine] 行为节点手册 (行为节点.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: Action Node, Blackboard, BTStatus, 移动/攻击/召唤/资源调度内置节点  
> **主要目标**: 提供行为树叶子节点（Action）的开发模板、黑板机制说明以及全量内置行为节点 API 速查。

---

## 1. 概述 (Overview)

行为节点是行为树的“执行者”（叶子节点）。
*   **职能**: 负责具体功能的实现，如移动、攻击、播放动画、修改数据。
*   **状态反馈**: 每一帧必须返回执行状态（SUCCESS / FAILURE / RUNNING）。

---

## 2. 黑板机制 (Blackboard)

黑板是节点间共享数据的容器。`StarkBTComponent` 在初始化时会自动注入以下核心引用：

| 键名 | 含义 | 备注 |
| :--- | :--- | :--- |
| `entityId` | 当前绑定的实体唯一 ID | - |
| `component` | 当前 `StarkBTComponent` 实例 | 可调用组件方法 |
| `factory` | MODSDK 引擎工厂 | - |
| `asset` | 内置 `StarkLevelAssetsComponent` 节点 | 处理资源 |
| `world` | 内置 `world` 节点 | 召唤、世界级 API |
| `molang` | 内置 `molang` 节点 | 修改渲染变量 |

## 3. 自定义行为节点模板

```python
from ..StarkEngine.StarkServer import Action, BTStatus

class MyCustomAction(Action):
    def on_start(self, blackboard):
        # 行为开始时的初始化
        self._ticks = 0

    def on_update(self, blackboard):
        # 每一帧的执行逻辑
        self._ticks += 1
        if self._ticks < 30:
            return BTStatus.RUNNING # 继续执行
        return BTStatus.SUCCESS     # 执行成功，结束
        
    def on_stop(self, blackboard):
        # 行为结束（成功/失败/被中断）后的清理
        pass
```

## 4. 内置行为节点 API 速查

### 4.1 通用控制
*   **LogAction(message)**: 打印调试日志（正式版建议删除）。
*   **WaitAction(duration)**: 固定时长等待。
*   **RandomWaitAction(min, max)**: 随机时长等待。
*   **SetBlackboardValue(key, value)**: 写入数据到黑板。
*   **RemoveBlackboardValue(key)**: 清除黑板数据。

### 4.2 移动与定位 (Movement)
*   **MoveToTarget(speed, maxIteration)**: 向当前攻击目标寻路。
*   **MoveToPosAction(pos, speed)**: 导航移动到指定世界坐标。
*   **TeleportToTarget(offset, randomHorizon, randomVertical)**: 瞬移到目标附近（支持随机偏移）。
*   **FacingToTarget()**: 强制朝向当前目标。
*   **StayStatic()**: 保持静止。

### 4.3 目标管理 (Targeting)
*   **ScanTarget(engineTypeStrWeight, range, distanceOrder)**: 扫描并设定攻击目标。
*   **ClearTarget()**: 清除当前攻击目标。

### 4.4 攻击手段 (Combat)
*   **MeleeAttack(damage, attack_range)**: 范围内单体近战伤害。
*   **BoxAttack(damage, box_size, box_offset)**: 面向方向的 AABB 盒状群体伤害。
*   **FireProjectile(engineTypeStr, posOffset, power, gravity, damage, traceTarget)**: 按朝向发射投射物。
*   **FireProjectileTowardsTarget(...)**: 向目标精准发射投射物。

### 4.5 召唤系统 (Summon)
*   **SummonEntity(summon_choices)**: 按权重、数量、光照等条件在附近生成实体。
*   **SummonEntityInStructure(summon_choices)**: 在特定方块结构（如水面）附近生成。
*   **SummonEntityAtOffset(entityIdentifier, posOffset, spreadRange)**: 固定偏移+随机扩散生成。

### 4.6 资源调度 (Render & Sound)
*   **PlayAnimation(animName, ac_name, stop_molang, anim_duration)**: 播放动画。
*   **PlayParticle(particleName, pos, Rotation, bindEntity, bindBone, nickName)**: 播放粒子（支持绑定骨骼）。
*   **RemoveParticle(nickName)**: 移除指定昵称的粒子。
*   **PlaySound(soundName, pos, volume, pitch, loop, nickName)**: 播放音效。
*   **StopSound(nickName)**: 停止指定音效。

### 4.7 环境与事件
*   **CreateFakeBlockArea(...)**: 在区域内方块上方生成持久化的假方块。
*   **Event(eventName)**: 触发实体的 DataDriven 自定义事件。
*   **SetMolangValue(queryName, value)**: 修改实体 Molang 值并存盘。
*   **AddMotion(velocity)**: 施加瞬时动量。

## 5. 组合逻辑示例

```python
# 逻辑：发现玩家 -> 走过去 -> 近战攻击
Sequence([
    ScanTarget({'minecraft:player': 100}, range=24),
    MoveToTarget(speed=1.0),
    MeleeAttack(damage=3, attack_range=2.5),
])

# 逻辑：有目标则射击，否则随机游走
Selector([
    Sequence([
        HasTarget(),
        FireProjectileTowardsTarget('minecraft:arrow', power=1.2)
    ]),
    RandomWaitAction(1, 2)
])
```
