# [StarkEngine] 行为树组合节点 (组合节点.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: Composite Node, Selector, Sequence, Reactive, Parallel, 逻辑分叉, 优先级抢占  
> **主要目标**: 深入介绍行为树的“枝干”——组合节点，理解其如何控制逻辑的流向、优先级以及并行处理。

---

## 1. 概述 (Overview)

组合节点是行为树的控制核心（枝节点）。它定义了子节点（行为或条件）的**执行顺序**与**决策逻辑**。根据是否具有“记忆”以及评估频率，SE 提供了五种核心组合节点。

---

## 2. 组合节点类型详解

### 2.1 Selector (选择器) —— “择优/兜底”
从上到下依次评估子节点，**只要有一个成功，整个 Selector 立即返回成功**，不再执行后续节点。

*   **逻辑特性**: 类似于 `if ... elif ... else`。
*   **典型用法**: 多分支择一执行、逻辑兜底。
*   **示例**:
    ```python
    Selector([
        Sequence([RandomChance(0.5), LogAction('50%几率成功')]),
        Sequence([RandomChance(0.3), LogAction('30%几率成功')]),
        LogAction('兜底逻辑：前两个都没中') # 100% 执行
    ])
    ```

### 2.2 Sequence (顺序器) —— “步骤链/条件检查”
从上到下依次执行子节点，**只有全部成功，整个 Sequence 才成功**。一旦某个节点失败，后续节点立即终止。

*   **逻辑特性**: 类似于 `if A and B and C`。
*   **典型用法**: 动作序列（先走再打）、条件检查（满足 A 且满足 B 才做 C）。
*   **示例**:
    ```python
    Sequence([
        HasTarget(),          # 条件1：有目标
        IsTargetInRange(0, 5),# 条件2：在攻击范围
        AttackAction()        # 执行：攻击
    ])
    ```

### 2.3 ReactiveSequence (反应式顺序) —— “条件敏锐型”
逻辑与 `Sequence` 类似，但**不具备记忆功能**。每一帧都会重新从第一个子节点开始 tick。

*   **核心痛点**: 解决 `Sequence` 运行长任务时忽略前置条件变化的问题。
*   **典型场景**: 目标中途消失、Buff 结束等需要立即打断后续任务的情况。
*   **示例**:
    ```python
    ReactiveSequence([
        HasTarget(),           # 每帧必检，目标消失则立即打断后面的移动
        MoveToTarget(speed=1.0)
    ])
    ```

### 2.4 ReactiveSelector (反应式选择) —— “优先级抢占型”
逻辑与 `Selector` 类似，但每一帧都会重新评估所有子节点，前方的分支具有**绝对高优先级**。

*   **逻辑特性**: 抢占式。如果高优先级的节点突然满足条件（SUCCESS 或 RUNNING），会立即打断（abort）正在执行的低优先级节点。
*   **典型场景**: 紧急撤退优先级高于追击、受到攻击立即防御。
*   **示例**:
    ```python
    ReactiveSelector([
        Sequence([IsUnderAttack(), Flee()]),   # 高优先级：挨打必跑
        Sequence([HasTarget(), ChaseTarget()]) # 低优先级：没打我时追杀目标
    ])
    ```

### 2.5 Parallel (并行节点) —— “多线程模拟”
在同一帧内对所有子节点执行 tick，根据 **策略 (Policy)** 决定整体结果。

*   **策略常量**:
    *   `Parallel.REQUIRE_ALL_SUCCESS` (默认): 全部成功才算成功；任一失败则整段失败。
    *   `Parallel.REQUIRE_ONE_SUCCESS`: 任一成功即成功；全部失败才算失败。
*   **典型场景**: 同时播放动画与音效、一边移动一边计时。
*   **示例**:
    ```python
    Parallel([
        PlayAnimation('animation.attack'),
        WaitAction(1.0) # 动画与等待同时跑，都成功才算完
    ], policy=Parallel.REQUIRE_ALL_SUCCESS)
    ```

## 3. 对照表速查 (Comparison Table)

| 节点类型 | 成功条件 | 失败条件 | 记忆(Memory) | 典型用途 |
| :--- | :--- | :--- | :--- | :--- |
| **Sequence** | 全部子节点成功 | 任一失败 | **有**（从当前继续） | 固定步骤、多重条件判断 |
| **Selector** | 任一子节点成功 | 全部失败 | **有** | 分支择一、最后兜底 |
| **ReactiveSequence** | 全部成功 | 任一失败 | **无**（每帧从头） | 条件易变的持续性任务 |
| **ReactiveSelector** | 任一成功 | 全部失败 | **无**（每帧从头） | 高优先级逻辑抢占/打断 |
| **Parallel** | 见 Policy | 见 Policy | - | 动画/音效/移动等多路并行 |

## 4. 组合使用示例

```python
tree = Selector([
    # 第一优先级：追击目标
    Sequence([
        HasTarget(),
        MoveToTarget(speed=1.2)
    ]),
    # 第二优先级/兜底：闲逛
    Sequence([
        RandomWaitAction(1, 2)
    ])
])
```
