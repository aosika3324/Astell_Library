## 1. 脚本目录结构规范 (Script Directory)

在 **行为包 (Behavior Pack)** 的脚本目录下，必须严格区分 **引擎本体** 与 **Mod 逻辑**，其层级关系如下：

```text
脚本目录/
├── StarkEngine/            # StarkEngine 引擎本体（核心目录）
│   ├── __init__.py
│   └── SE_Core.bin         # 引擎核心二进制文件
├── Mod/                    # 当前项目的业务逻辑目录
│   ├── __init__.py
│   ├── rootClient.py       # 客户端根节点（场景组件）
│   ├── rootServer.py       # 服务端根节点（场景组件）
│   └── ...                 # 其他业务代码
├── __init__.py
└── modMain.py              # 【关键】Mod 入口文件，系统最先加载此处
```

---

## 2. Mod 入口程序 (`modMain.py`)

`modMain.py` 是整个 Mod 的启动引擎，负责加载 StarkEngine 环境并挂载双端逻辑根节点。

### 2.1 初始化代码模板

```python
# -*- coding: utf-8 -*-
from .StarkEngine.Stark import *       # 导入 StarkEngine 引擎主类
from .Mod.rootClient import rootClient # 导入项目客户端根节点
from .Mod.rootServer import rootServer # 导入项目服务端根节点

# 1. 实例化引擎对象
mod = StarkEngine()

# 2. 指定服务端根节点 (Server Scene Component)
mod.ServerRootNode(rootServer)

# 3. 指定客户端根节点 (Client Scene Component)
mod.ClientRootNode(rootClient)
```

### 2.2 核心要点说明
*   **加载顺序**: 游戏引擎会首先扫描并执行脚本目录下的 `modMain.py`。
*   **根节点挂载**: `mod.ServerRootNode` 和 `mod.ClientRootNode` 是连接引擎核心与业务逻辑的桥梁。
*   **场景组件化**: `rootServer` 和 `rootClient` 通常被视为整个项目的逻辑容器（场景组件）。
