# [StarkEngine] 组件系统开发手册 (组件.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: ECS, 生命周期, 节点操作, 双端变量同步, 持久化定时器  
> **主要目标**: 提供 StarkEngine 组件 (Component) 的全周期管理与功能接口说明。

---

## 1. 组件概述与分类

组件是 StarkEngine 中具有特定生命周期的功能对象，主要分为以下三类：

| 类型 | 类名 (Class) | 挂载需求 | 场景描述 |
| :--- | :--- | :--- | :--- |
| **场景组件** | `StarkBaseLevelComponent` | 无需挂载 | 存档级全局逻辑，如天气、全局规则 |
| **实体组件** | `StarkBaseActorComponent` | 挂载到生物实体 | 针对特定生物的逻辑，如 AI 行为、属性加成 |
| **方块组件** | `StarkBaseBlockEntityComponent` | 挂载到方块实体 | 针对方块实体的逻辑，如熔炉、容器逻辑 |

### 核心特性
*   **引擎管理**: 所有组件由 SE 引擎统一管理，生命周期严格受控。
*   **自动生命周期**: 实体组件随宿主死亡/出入区块自动销毁与重新加载。

---

## 2. 生命周期 (Lifecycle)

组件生命周期由引擎内部严格驱动，主要包含三个核心节点：

1.  **Start**: 组件开始运行时首先执行。
2.  **Tick**: 运行中持续执行，频率为 **30帧/秒**。
3.  **End**: 组件即将被销毁前执行。

---

## 3. 组件管理 API (Node Operations)

### 3.1 增删改查
*   **AddNode(nodeName, nodeCls)**: 新增组件。  
    `self.AddNode('MyNode', MyComponentClass)`
*   **GetNodeByName(nodeName)**: 根据名称获取场景/宿主中的组件。  
    `node = self.GetNodeByName('MyNode')`
*   **GetNodes()**: 获取场景/宿主中所有的组件。
*   **RemoveNodeByName(nodeName)**: 根据名称销毁指定组件。
*   **Remove()**: 销毁当前组件自身。

### 3.2 静态类型转换
*   **CastTo(cls)**: 将获取到的组件类型转换为目标类。  
    **目的**: 获得 IDE 代码补全支持。  
    `self.GetNodeByName('MyNode').CastTo(MyNodeClass)`

## 4. 环境判断与辅助功能

### 4.1 逻辑帧判断
*   **IsLogicTick()**: 判断当前 Tick 是否为原生的 20 帧逻辑帧（剔除网易魔改的 10 帧）。
*   **GetRunSide()**: 返回当前运行端（`"Client"` 或 `"Server"`）。
    *   `isServer = self.GetRunSide() == "Server"`

## 5. 变量同步与监听

### 5.1 双端变量同步 (仅服务端可用)
*   **SetSyncServerVar(clientVarName, serverVarName)**:  
    当服务端的变量改变时，自动将值同步到客户端对应的变量上。

```python
# 服务端
def __init__(self, *args, **kwargs):
    self.sync_var = 1
    self.SetSyncServerVar('sync_var_client', 'sync_var') # 自动同步

# 客户端
def __init__(self, *args, **kwargs):
    self.sync_var_client = 1 # 初始值
```

### 5.2 变量监听回调
*   **SetOnVarChange(varName, callback)**:  
    监听当前类成员变量的改变，并触发回调。
    *   `self.SetOnVarChange('test_var', self.OnTestVarChange)`
    *   回调签名: `def OnTestVarChange(self, newValue):`

## 6. 定时器系统 (Timer)

### 6.1 基础定时器 (随组件生命周期结束)
*   **AddTimer(time, callback, repeat=False, nickName=None, *args, **kwargs)**:  
    支持传参、循环及昵称覆盖。
    *   `self.AddTimer(1, self.MyCallback, False)`

### 6.2 游戏刻持久化定时器 (存档持久化)
*   **重要提示**: 必须由 **Root 组件** 驱动。
*   **特性**: 重启存档后计时依然存在，无需重新添加。

| 接口 | 描述 |
| :--- | :--- |
| **AddGameTimeTimer** | 添加游戏刻定时器（回调必须为成员方法）。 |
| **RemoveGameTimeTimer** | 根据昵称移除。 |
| **GetGameTimeTimer** | 获取定时器字典信息。 |
| **IsAlreadyHasGameTimeTimer** | 检查是否存在。 |
