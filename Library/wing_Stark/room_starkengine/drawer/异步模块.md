# [StarkEngine] 异步模块 (异步模块.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: Async, @AsyncMethod, yield sleep, async_request, RPC 通信  
> **主要目标**: 介绍如何在网易版单线程环境下通过类 Promise 设计实现非阻塞延迟与双端异步数据交互。

---

## 1. 为什么需要异步模块？

网易我的世界基岩版是**单线程运行**的。这意味着：
*   **严禁使用 Python 原生 `sleep`**: 否则主线程会直接暂停，导致游戏卡死、崩溃。
*   **传统回调繁琐**: 双端通信（C/S 架构）通常需要多次监听、指定回调、配合 Timer，导致代码难以维护且逻辑断层。

**StarkEngine 解决方案**: 提供一套类 JavaScript Promise 设计的异步系统，支持使用 `yield` 挂起协程。

---

## 2. 基础用法与核心装饰器

### 2.1 `@AsyncMethod` (必选)
这是开启异步功能的唯一开关。只有被该装饰器标记的函数才支持异步流程控制。
*   **约束条件**: 被标记的函数**严禁使用 `return`** 返回任何数据（数据通过 `yield` 的结果获取）。

### 2.2 非阻塞延迟 (`sleep`)
在异步函数中，使用 `yield sleep(duration)` 实现延迟，期间主线程会正常运行，游戏不会卡顿。

**代码示例**:
```python
@AsyncMethod
def TestAsyncFunc(self):
    print('延迟 5 秒开始')
    yield sleep(5)  # 协程挂起，主线程不卡
    print('延迟 5 秒结束')
```

## 3. 双端远程通信

### 3.1 异步请求 (`async_request`)
用于在某一端调用另一端的函数，并**等待其执行结果**。
*   **超时机制**: 若 5 秒内未获取到返回结果，则判定为请求失败，代码继续向下执行。

**代码示例 (客户端调用服务端)**:
```python
# 客户端
@AsyncMethod
def GetServerData(self):
    # 向名为 'TestServerNode' 的组件请求 'GetSomeValue' 函数，参数为 (1, True)
    result = yield async_request('TestServerNode', 'GetSomeValue', 1, True)
    
    isSuccess = result[0] # 本次请求是否成功
    if isSuccess:
        content = result[1] # 获取函数实际的 return 值
```

### 3.2 常规远程请求 (`request`)
经典的 RPC 调用，仅负责触发另一端的函数执行，**不关心返回值**。不需要 `@AsyncMethod` 支持。

**代码示例**:
```python
# 直接触发远程函数
request('TestServerNode', 'DoSomethingWithoutReturn', param1)
```

---

## 4. 接口对比总结

| 功能 | 接口 | 是否需要 `@AsyncMethod` | 是否阻塞协程 | 场景建议 |
| :--- | :--- | :--- | :--- | :--- |
| **延时** | `yield sleep(n)` | **是** | 是 (挂起协程) | 简单的延时执行逻辑 |
| **异步请求结果** | `yield async_request(...)` | **是** | 是 (等待结果) | 需要获取另一端数据后再继续逻辑 |
| **单向 RPC** | `request(...)` | 否 | 否 (立即执行) | 仅需通知另一端执行操作 |
