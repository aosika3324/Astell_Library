# [StarkEngine] 事件监听手册 (事件监听.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: SubscribeEvent, 事件驱动, 现代化监听, 自动参数映射  
> **主要目标**: 介绍 StarkEngine 中如何通过装饰器实现高效、类型友好的事件监听，解决原生 args['xxxx'] 字典取值效率低下的问题。

---

## 1. 概述 (Overview)

SE 提供了一套现代化且便捷的事件监听逻辑，旨在解决网易原生接口依赖文档、字典取值繁琐且易错的痛点。

### 核心优势
*   **对象化参数**: 通过 `event.xxxx` 直接获取事件参数，支持代码补全提示。
*   **简洁语法**: 使用 `@SubscribeEvent` 装饰器即可完成注册，无需手动维护监听列表。
*   **支持范围**: 此监听方式目前仅支持 **组件类 (Components)** 与 **UI 类 (UI Classes)**。

---

## 2. 用法示例 (Usage)

通过导入 `Events` 预定义类并配合装饰器实现监听。

```python
from ..StarkEngine.StarkServer import * # 或 StarkClient

class MyCustomComponent(StarkBaseLevelComponent):
    def __init__(self, *args, **kwargs):
        StarkBaseLevelComponent.__init__(self, *args, **kwargs)
    
    # 1. 使用装饰器标记监听函数
    # 2. 设置 event 参数的默认值为对应的事件对象，以实现类型映射
    @SubscribeEvent
    def onPlayerDie(self, event=Events.PlayerDieEvent()):
        """玩家死亡事件回调"""
        # 直接通过 .id 获取，无需 args['id']
        playerId = event.id
        print(f"检测到玩家死亡，ID 为: {playerId}")
```

---

## 3. 核心机制说明

1.  **自动注册**: 引擎在实例化组件或 UI 时，会自动扫描带有 `@SubscribeEvent` 装饰器的成员方法并执行底层监听绑定。
2.  **参数解构**: 监听函数中的 `event` 对象会自动承载引擎下发的数据字段。
3.  **开发建议**: 始终为监听函数提供带有默认事件实例的 `event` 参数（如 `event=Events.XxxEvent()`），这不仅是引擎识别事件类型的依据，也能为 IDE 提供强大的静态分析与补全支持。
