# [StarkEngine] 方块组件 (方块组件.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: StarkBaseBlockEntityComponent, 方块实体, RegisterBlockEntityProxyComponent, 数据持久化, Molang 同步  
> **主要目标**: 介绍如何针对“方块实体 (BlockEntity)”开发组件，并掌握其独特的数据存储与操作接口。

---

## 1. 概述 (Overview)

方块组件继承自 `StarkBaseBlockEntityComponent`，专门用于绑定游戏中的 **方块实体 (BlockEntity)**。

### 1.1 前置概念
*   **绑定对象**: 必须是方块实体，而非普通方块。普通方块不具备存储复杂数据或挂载逻辑的能力。
*   **创建机制**: 方块组件**不使用**通用的 `AddNode` 接口创建。
*   **统一管理**: 必须通过 SE 的全局统一函数进行注册与管理。详见：`StarkEngine/模块/方块组件/RegisterBlockEntityProxyComponent`。

---

## 2. 数据操作与持久化 (Data Management)

方块组件提供了比实体组件更直接的 KV 数据存储能力。

### 2.1 核心数据接口
*   **SetData(key, value)**: 设置并保存数据，规则遵循网易接口。
*   **GetData(key, default)**: 获取数据，若键名不存在则返回默认值。
    *   `type = self.GetData('cooking_type', 'none')`
*   **GetAllData()**: 返回该方块实体上存储的所有数据字典。

### 2.2 Molang 联动与同步
*   **SetMolang(key, value)**: 设置 Molang 变量。**注意：** 此接口设置的值不会被保存，重新加载区块后会丢失。
*   **Molang 自动持久化 (推荐)**: 
    *   当 `SetData` 的 key 以 `molang_` 开头时（如 `molang_abc`），SE 不仅会保存该变量，还会自动同步设置 `variable.xxx`。
    *   **优势**: 方块在区块重新加载时会自动恢复这些 Molang 值。

## 3. 方块空间与属性查询

| 方法 | 描述 |
| :--- | :--- |
| **GetBlockName()** | 获取当前组件挂载方块的标识符 (Identifier)。 |
| **GetFacingRotY()** | 获取方块的水平朝向角度（仅对具备朝向属性的方块有效）。 |
| **GetNeighborBlock(face)** | 获取指定面 (Face) 的相邻方块坐标。Face 取值见网易枚举。 |
| **DestroySelf()** | 破坏当前组件挂载的方块实体，组件随之销毁。 |

---

## 4. 开发注意事项
*   **生命周期**: 当方块被破坏时，关联的方块组件会自动触发 `End` 并销毁。
*   **性能建议**: 对于需要频繁在客户端渲染表现（通过 Molang）且需要存档保存的数据，请务必使用 `SetData('molang_xxx', value)` 格式。
