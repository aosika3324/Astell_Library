## 1. 概述 (Overview)

`StarkLevelBlockComponent` 是一个全局单例的场景组件，用于对存档世界中的方块执行底层操控。

### 核心特性
*   **获取方式**: 
    `blockNode = self.GetNodeByName(StarkNodes.Block)`
*   **核心功能**: 销毁/替换方块、持久化数据读写、Molang 渲染同步、自动代理逻辑注册。

---

## 2. 方块物理操作

| 接口 | 描述 | 关键参数 |
| :--- | :--- | :--- |
| **DesroyBlock** | 销毁指定位置的方块。 | `position`, `dimensionId` |
| **ReplaceBlock** | 替换方块，支持冷却机制。 | `newBlockDict` (含 name 和 aux), `cd` (Tick，默认 1) |
| **GetBlockName** | 获取方块的 Identifier 标识符。 | `position`, `dimensionId` |
| **GetNeighborBlock** | 获取相邻方块的维度与坐标。 | `facing` (0:下, 1:上, 2:北, 3:南, 4:西, 5:东) |
| **GetFacingRotY** | 获取具备朝向属性方块的 Y 轴角度。 | `position`, `dimensionId` |

## 3. 方块数据持久化与 Molang 联动

这是实现“机器进度条”、“容器状态”等功能的核心。

### 3.1 核心数据接口
*   **SetData(dimensionId, position, key, value)**:
    *   **持久化**: 将数据保存至对应方块实体的 NBT 中。
    *   **Molang 同步**: 若 `key` 以 `molang_` 开头，SE 会自动将其同步为方块的 Molang 渲染变量（`variable.xxx`）。
*   **GetData(dimensionId, position, key, default)**: 读取指定方块上的持久化数据。
*   **GetAllData(dimensionId, position)**: 返回方块上所有的自定义数据字典。

### 3.2 运行时渲染控制
*   **SetMolang(dimensionId, position, molangName, value)**:
    *   **仅运行时生效**：用于瞬时的渲染状态改变。
    *   **注意**：此方法**不存档**，重新加载区块后 Molang 值会重置。

## 4. 方块实体代理 (Proxy) 机制

SE 通过代理模式实现了方块逻辑的**自动激活**。

### 4.1 核心原理
当你为一个方块标识符注册了代理组件后，场景中凡是放置该标识符的方块实体，都会被自动挂载一个对应的组件实例来管理其业务逻辑。

### 4.2 注册接口
*   **RegisterBlockEntityProxyComponent(blockIdentifier, componentClass)**
    *   `blockIdentifier`: 目标方块标识符 (如 `myapp:machine`)。
    *   `componentClass`: 代理组件类 (必须继承自 `StarkBaseBlockEntityComponent`)。

**代码示例**:
```python
# 在初始化阶段注册逻辑绑定
self.RegisterBlockEntityProxyComponent("myapp:furnace", MyFurnaceLogicComponent)
```
