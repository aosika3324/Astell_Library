# [StarkEngine] 世界功能组件 (世界功能组件.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: 实体生成, 结构匹配, 坐标找点, 地表探测, 服务端逻辑  
> **主要目标**: 提供一套在服务端按规则、按结构进行批量实体生成的封装方案。

---

## 1. 概述 (Overview)

世界功能组件是一个专门用于服务端逻辑的**场景组件**。它高度封装了“找点”与“生成”逻辑，支持按方块结构匹配、光照区间过滤以及自定义流程筛选。

---

## 2. 实体生成接口 (Spawn APIs)

### 2.1 按竖直结构匹配生成 (`SpawnEntityAtPosInStructure`)
在锚点周围随机落点，并按给定的“竖直方块序列”进行环境匹配。
*   **THIS 占位符**: 代表生成点所在的那一列参考格。
*   **适用场景**: 水面漂浮物、洞穴顶部挂载物、特定地层生物。
*   **示例**:
    ```python
    # 逻辑：脚下是 THIS (参考格)，头顶是空气，再往上两格是水
    structure = ['minecraft:air', 'THIS', 'minecraft:water', 'minecraft:water']
    self.SpawnEntityAtPosInStructure('my:boat', pos, dim, gen_range=[0, 10], count=[1, 3], structure=structure)
    ```

### 2.2 按自定义流程生成 (`SpawnEntityAtPosInProcess`)
通过回调函数 `processFunc` 决定坐标是否合法。
*   **processFunc**: `(rx, ry, rz, dimensionId) -> (x, y, z) or None`。
*   **适用场景**: 复杂的条件过滤（如：必须在玩家视线内生成）。

### 2.3 基础地表生成 (`SpawnEntityAtPos`)
最常用的陆地刷怪/植物生成方案。
*   **核心逻辑**: 随机落点 $\rightarrow$ `GetSurfaceBlock` 找地表 $\rightarrow$ 光照过滤 $\rightarrow$ 生成。
*   **关键参数**: `lightDemand` [low, high]（光照区间）、`targetBlocks`（目标脚下方块）。

## 3. 找点与探测工具 (Utility)

| 方法 | 描述 | 返回值 |
| :--- | :--- | :--- |
| **GenerateRandomPos** | 以锚点为中心，在固定 Y 轴进行水平圆环随机。 | `(x', y, z')` |
| **GetTopBlock** | 获取指定列 (x, z) 的最高非空方块高度。 | `(x, y, z)` |
| **GetSurfaceBlock** | 按 `targetBlock` (脚下) 和 `passList` (头上) 条件向下扫描地表。 | `(x, y, z)` |
| **RetFirstAirBlockPos** | 从基准点随机向上或向下探测，寻找第一个可用的空气位。 | `(x, y, z)` |

## 4. 开发选型速查表

| 需求场景 | 推荐方法 |
| :--- | :--- |
| **水柱、洞穴口等特定竖直环境匹配** | `SpawnEntityAtPosInStructure` |
| **需要编写复杂 Python 条件过滤** | `SpawnEntityAtPosInProcess` |
| **常规陆地生物生成（含地表+光照限制）** | `SpawnEntityAtPos` |
| **寻找列最高点** | `GetTopBlock` |
| **寻找标准地表（如草地/泥土上）** | `GetSurfaceBlock` |
| **基准点附近快速找空气位/垫高** | `RetFirstAirBlockPos` |

---

## 5. 使用提示
1.  **服务端专属**: 所有世界功能组件的方法必须在服务端调用。
2.  **性能注意**: 批量生成（`count` 区间较大）及复杂的 `processFunc` 可能会在短时间内消耗服务器性能，请合理设置生成频率。
3.  **THIS 语义**: 在 `structure` 列表中，`THIS` 的位置决定了其他方块相对于生成点的竖直偏移方向（列表顺序即竖直关系）。
