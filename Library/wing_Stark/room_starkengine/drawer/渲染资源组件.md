# [StarkEngine] 渲染资源管理组件 (渲染资源组件.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: StarkNodes.Render, 生物渲染, 玩家渲染, 状态机控制, 批量配置  
> **主要目标**: 提供 StarkEngine 中对于我的世界全部渲染资源类型（动画、模型、贴图、材质等）的统一管理接口。

---

## 1. 概述 (Overview)

渲染资源组件是一个专门用于控制游戏中各种渲染表现的**场景组件**。

### 核心特性
*   **单例模式**: 该组件在双端各自保持单例。
*   **获取方式**: 
    `renderNode = self.GetNodeByName(StarkNodes.Render)`
*   **重要约束**: **若需修改玩家类型的渲染方案，必须通过服务端接口发起请求！**

---

## 2. 生物渲染操作 (Actor Render)

针对指定生物标识符 (`entityIdentifier`) 层面进行的全局渲染配置。

### 2.1 核心配置接口
*   **add_actor_geometry**: 为指定生物添加几何体模型。
*   **add_actor_texture**: 为指定生物添加贴图。
*   **add_actor_material**: 为指定生物添加材质。
*   **add_actor_particle**: 为指定生物添加粒子效果。
*   **add_actor_sound**: 为指定生物添加音效。

### 2.2 动画控制
*   **add_actor_animation**: 添加基础动画。
*   **add_actor_animation_controller**: 添加动画控制器 (AC)。
*   **add_actor_animation_into_state**: 精准控制。为指定 AC 的特定 **状态 (State)** 添加动画，并支持 **Molang 条件判断**。
*   **add_actor_script_animate**: 添加脚本动画 (Animate)，支持自动替换逻辑。

### 2.3 渲染控制器
*   **add_actor_render_controller**: 添加渲染控制器 (RC)，支持 Molang 触发条件。
*   **add_actor_render_controller_array**: 以数组形式批量注入渲染控制逻辑。

---

## 3. 玩家渲染配置 (Player Render)

针对特定玩家 (`playerId`) 的渲染控制。若 `playerId` 传入 `-1`，则应用为**全局默认配置**（为所有玩家添加）。

### 3.1 玩家接口列表 (Player-only)
*   `add_player_geometry` / `add_player_texture` / `add_player_material`
*   `add_player_animation` / `add_player_animation_controller`
*   `add_player_animation_into_state` (状态机内插动画)
*   `add_player_particle` / `add_player_sound`
*   `add_player_render_controller` / `add_player_script_animate`

---

## 4. 单体与精准渲染 (Single Entity Render)

针对场景中**特定的某一个**实体 ID (`entityId`) 进行的精细化渲染控制。

| 接口 | 描述 |
| :--- | :--- |
| **add_actor_material_for_one** | 仅为当前这一个实体添加材质。 |
| **add_actor_particle_for_one** | 仅为当前这一个实体添加粒子效果。 |
| **reset_one_actory_render** | 重置该实体的渲染配置，恢复初始状态。 |
| **copy_player_material_to_entity** | 将玩家 A 的材质配置直接拷贝给实体 B。 |

---

## 5. 批量渲染配置方案 (Batch Config)

为了提高开发效率，支持通过 Python 字典一次性注入多项属性。

*   **add_actor_render_config(entityIdentifier, renderConfig)**: 批量配置生物。
*   **add_player_render_config(playerId, renderConfig)**: 批量配置玩家。

> **注**: `renderConfig` 字典的键名请参考引擎内部定义的 `RenderConfigKey` 枚举。
