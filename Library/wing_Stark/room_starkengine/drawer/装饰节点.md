# [StarkEngine] 装饰节点手册 (装饰节点.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: Decorator Node, Inverter, AlwaysSucceed, Repeat, 结果转换, 循环控制  
> **主要目标**: 介绍行为树中的“单孩子包装层”——装饰节点，理解如何通过装饰器改变子节点的运行结果或执行频率。

---

## 1. 概述 (Overview)

装饰节点（Decorator）是行为树的包装层。
*   **结构特性**: 只能拥有**唯一一个**子节点。
*   **职能**: 不改变子节点的本质逻辑，仅改变其返回值的含义（如取反）或执行次数（如循环）。
*   **基类**: `Decorator(BTNode)`。

---

## 2. 核心装饰节点类型

### 2.1 Inverter (取反器) —— “非逻辑”
对子节点的结果进行逻辑对调。
*   **映射关系**:
    *   `SUCCESS` -> `FAILURE`
    *   `FAILURE` -> `SUCCESS`
    *   `RUNNING` -> `RUNNING` (保持不变)
*   **典型场景**: “如果没有目标则执行”。
*   **示例**: `Inverter(HasTarget())`

### 2.2 AlwaysSucceed (恒通装饰器) —— “容错/强制成功”
无论子节点执行结果如何（只要不是 `RUNNING`），一律向上层返回 `SUCCESS`。
*   **典型场景**: `Sequence` 中某个非核心步骤失败时不应中断整条链。
*   **示例**: `AlwaysSucceed(WalkToRandomPos(radius=[2, 8]))`

### 2.3 Repeat (重复执行器) —— “循环控制”
让子节点重复执行指定的次数。
*   **参数**:
    *   `child`: 目标子节点。
    *   `count`: 重复次数。`-1` 表示无限循环。
*   **运行规则**: 每次子节点返回 `SUCCESS` 后，装饰器会调用 `child.reset()` 并进入下一轮。
*   **注意**: 
    *   若 `count=-1` 且子节点总是瞬时成功，行为树会“卡”在这一步，需配合外层 `Reactive` 节点打断。
    *   单轮 `SUCCESS` 后，当帧会返回 `RUNNING` 以避免同帧死循环。

---

## 3. 装饰器速查表

| 类名 | 功能简述 | 结果干预 | 典型用途 |
| :--- | :--- | :--- | :--- |
| **Inverter** | 结果取反 | S $\leftrightarrow$ F | 实现 "Not" 逻辑判断 |
| **AlwaysSucceed** | 强制成功 | F $\rightarrow$ S | 容错处理、防止序列中断 |
| **Repeat** | 循环执行 | 累计次数 | 连招、持续性动作、循环检查 |

## 4. 组合使用示例

### 4.1 连招逻辑
```python
# 有目标且在范围内时，连续尝试 2 次近战攻击
Sequence([
    HasTarget(),
    IsTargetInRange(1.0, 6.0),
    Repeat(
        MeleeAttack(damage=2, attack_range=2.5),
        count=2
    )
])
```

### 4.2 非逻辑判断
```python
# 仅当实体“未处于眩晕状态”时，装饰器返回 SUCCESS
Inverter(
    CheckBlackboardValue('stunned', True)
)
```

### 4.3 容错选择
```python
# 如果追击失败，强制通过“随机走动”分支，保证逻辑不卡死
Selector([
    Sequence([HasTarget(), ChaseTarget()]),
    AlwaysSucceed(WalkToRandomPos())
])
```
