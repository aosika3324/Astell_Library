# [StarkEngine] 实体组件 (实体组件.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: StarkBaseActorComponent, 实体绑定, Bind, 生命周期共生, 区块加载/淡出  
> **主要目标**: 介绍如何将逻辑绑定至具体生物实体，并掌握实体组件的特殊加载与注销机制。

---

## 1. 概述与生命周期同步 (Overview)

实体组件继承自 `StarkBaseActorComponent`，是专门为生物/实体对象设计的组件类型。其生命周期与宿主实体（Actor）紧密绑定。

### 1.1 自动注销与销毁
*   **物理注销**: 实体的死亡、被引擎注销或从内存中删除，都会导致该组件自动执行 `End` 并销毁。
*   **客户端特性**: 
    *   **淡出区块**: 实体远离玩家视野或离开模拟区块时，客户端组件会**主动销毁**。
    *   **重新进入**: 当实体重新进入玩家视野或区块加载范围时，会**重新实例化并触发 Start**。

---

## 2. 关键激活流程：AddNode -> Bind

与场景组件不同，实体组件在创建后不会立即启动。

### 2.1 挂起状态
使用 `AddNode` 创建实体组件后，该组件虽然已实例化，但其 `Start()` 函数不会被触发，整个生命周期处于**停止（挂起）**状态。

### 2.2 绑定激活 (Bind)
必须通过 `Bind(entityId)` 方法将其与世界中具体的实体绑定后，才会正式触发 `Start` 并开始其周期。

**代码示例**:
```python
# 1. 实例化组件 (此时处于挂起状态，不触发 Start)
entityNode = self.AddNode('PigTestComp', MyEntityComponentClass)

# 2. 绑定目标实体 (此时正式触发 Start，生命周期开始)
entityNode.Bind('-1278230438') 
```

## 3. 实体管理 API

| 方法 | 描述 | 返回值 |
| :--- | :--- | :--- |
| **Bind(entityId)** | 将组件与指定实体 ID 进行强绑定，并激活生命周期。 | 无 |
| **GetEntityId()** | 获取当前组件所绑定的实体唯一 ID。 | `str` (实体ID) |
| **IsEntityInMemory()** | 检测当前绑定的实体是否仍在当前端的内存中。 | `bool` |

### 应用场景示例 (Start 阶段)
```python
def Start(self):
    self.entityId = self.GetEntityId()
    # 逻辑初始化时确认实体是否仍在内存中
    if self.IsEntityInMemory():
        print("实体已成功加载至内存并完成绑定")
```
