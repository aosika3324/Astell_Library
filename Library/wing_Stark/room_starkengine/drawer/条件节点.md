# [StarkEngine] 条件节点手册 (条件节点.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: Condition Node, check(), HasTarget, RandomChance, 逻辑决策  
> **主要目标**: 介绍行为树决策叶子节点（Condition）的开发模板、运行机制及内置条件节点的使用。

---

## 1. 概述 (Overview)

条件节点是行为树的“决策者”（叶子节点）。
*   **职能**: 仅负责“是/否”的逻辑判断，不执行任何实际的“行为”。
*   **运行特性**: 无 `RUNNING` 状态。
*   **逻辑映射**: 
    *   `True` -> `BTStatus.SUCCESS`
    *   `False` -> `BTStatus.FAILURE`

## 2. 自定义条件节点模板

```python
from ..StarkEngine.StarkServer import Condition

class MyPhaseCondition(Condition):
    """自定义条件节点：检查黑板中的阶段值"""
    
    def check(self, blackboard):
        # 核心逻辑：只做只读判断，避免副作用
        current_phase = blackboard.get('phase')
        return current_phase == 2
```

### 开发建议
1.  **只读原则**: `check` 方法内应仅执行读取操作，不建议修改世界状态或写入黑板数据。
2.  **基类能力**: 可通过基类方法 `GetLoaderSystem()` 或 `GetNodeByName(name)` 获取引擎环境。

## 3. 内置条件节点速查

| 类名 | 功能描述 | 参数说明 |
| :--- | :--- | :--- |
| **CheckBlackboardValue** | 判断黑板指定键的值是否等于预期值。 | `key`, `value` |
| **HasBlackboardValue** | 判断黑板是否存在指定键且值不为 `None`。 | `key` |
| **HasTarget** | 判断当前实体是否有有效的攻击目标。 | 无 |
| **IsTargetInRange** | 判断与当前目标的距离是否在 `[min, max]` 区间内。 | `min_dist`, `max_dist` |
| **RandomChance** | 按指定概率采样，决定本帧是否通过。 | `chance` (0.0~1.0) |

## 4. 组合使用示例

条件节点通常作为 `Sequence` 的前置门槛，用于保护后续行为逻辑。

```python
# 案例：只有满足以下三重条件，才会执行攻击并打印日志
Sequence([
    HasTarget(),               # 1. 必须有目标
    IsTargetInRange(2.0, 8.0), # 2. 目标必须在 2~8 格范围内
    RandomChance(0.4),         # 3. 40% 的概率通过（模拟反应迟钝或技能CD）
    LogAction('条件全部达成，执行动作'),
    AttackAction()             # 执行具体行为
])
```
