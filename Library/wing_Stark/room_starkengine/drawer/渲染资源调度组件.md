# [StarkEngine] 渲染资源调度组件 (渲染资源调度组件.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: 渲染资源调度, 动画播放, 粒子系统, 音效控制, 假方块渲染, nickName 别名机制  
> **主要目标**: 提供 StarkEngine 中对于动画、粒子、音效以及临时视觉反馈（假方块）的高效调度接口。

---

## 1. 概述与操作规则

渲染资源调度组件是一个负责“表现层”逻辑调度的场景组件。

### 核心特性
*   **双端单例**: 在客户端与服务端均保持单例运行。
*   **别名/ID 互换机制**: 在接口参数中，若同时存在 `pid` (实例 ID) 和 `nickName` (别名) 的缺省参数，代表**两者任传其一即可**完成操作。
*   **使用建议**: 优先传入稳定的 `nickName`，以便于后续逻辑中精确回收（停止）资源。

---

## 2. 核心调度接口

### 2.1 原生命令执行 (仅服务端)
*   **ExecuteCommand(target, commandStr)**: 直接在服务器执行 Minecraft 原生命令。

### 2.2 动画播放 (仅服务端)
*   **PlayAnimation(target, animName, ac_name, stop_molang, anim_duration, next_state)**:
    *   **默认 AC**: `stark_runtime_root`
    *   **默认停止条件**: `q.all_animations_finished` (Molang)

---

## 3. 粒子系统调度

StarkEngine 支持双版本粒子系统（微软原生与网易粒子）。

### 3.1 微软粒子 (Microsoft Particle)
| 接口 | 描述 | 备注 |
| :--- | :--- | :--- |
| **PlayParticle** | 创建并播放粒子。 | 可绑定实体/骨骼，支持指定别名。 |
| **RemoveParticle** | 销毁指定粒子。 | 通过 `nickName` 或 `pid` 定位。 |
| **SetParticleVariable** | 动态设置粒子变量。 | 调节渲染参数。 |
| **SetParticleVisible** | 控制显示/隐藏。 | - |
| **GetNickNameByPid** | 客户端专属，通过 PID 反查别名。 | - |

### 3.2 网易粒子 (NetEase Particle/VFX)
| 接口 | 描述 | 备注 |
| :--- | :--- | :--- |
| **createParticle** | 加载 vfx 路径并播放。 | 支持别名。 |
| **SetPlayOrStop** | 强制播放或停止。 | 通过 `nickName` 或 `pid` 定位。 |
| **BindEntity** | 绑定到实体。 | 支持偏移 (`offset`)、旋转及客户端实体判断。 |
| **SetPos** | 客户端专属。 | 手动同步粒子坐标。 |

---

## 4. 音效调度 (Sound)

*   **PlaySound(soundName, pos, nickName, volume, pitch, loop, entityId)**:
    *   支持 3D 坐标播放或绑定至特定实体。
    *   支持音量、音高及循环配置。
*   **StopSound(nickName, sid, fadeTime)**:
    *   支持设置 `fadeTime` (淡出时间)，默认 0.5 秒。

---

## 5. 假方块渲染 (Fake Block)

*   **GenFakeBlock(blockName, position, dimensionId, duration)**:
    *   **仅服务端可用**。
    *   在特定位置生成一个临时存在的方块实体，持续 `duration` 秒后自动销毁。
    *   **应用场景**: 击中落点反馈、技能指示器、警告标记预览等。

---

## 6. 开发性能建议
1.  **广播机制**: 双端接口在服务端调用时会广播至全体客户端，请避免在 `Tick` 等高频逻辑中无节制触发。
2.  **资源回收**: 定时销毁（如 `GenFakeBlock`）或手动回收（通过 `nickName` 移除粒子/停止音效）是防止内存泄漏的关键。
