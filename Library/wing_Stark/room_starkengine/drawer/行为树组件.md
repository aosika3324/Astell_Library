# [StarkEngine] 行为树组件 (行为树组件.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: BTComponent, BuildBehaviorTree, 实体行为树绑定, 逻辑封装  
> **主要目标**: 介绍如何通过继承 `BTComponent` 类来封装并运行自定义的 AI 行为树逻辑。

---

## 1. 概述 (Overview)

行为树组件 (`BTComponent`) 是 StarkEngine 行为树系统的运行载体。

### 核心定位
*   **组件类型**: 它本质上是一个**实体组件** (`StarkBaseActorComponent`)。
*   **运行前提**: 必须通过 `AddNode` 挂载并通过 `Bind` 绑定到具体的实体上，才能启动行为树逻辑。
*   **职能**: 负责定义、构建并持有该实体的行为树结构。

---

## 2. 核心开发接口: `BuildBehaviorTree`

在开发行为树组件时，**必须复写** `BuildBehaviorTree` 方法。

*   **作用**: 该函数用于描述行为树的层级结构（节点组合）。
*   **返回值**: 必须返回行为树的 **根节点 (Root Node)**。只有返回了根节点，SE 引擎才能接管并执行该树。

---

## 3. 代码实现模板

以下是一个完整的行为树组件类实现示例（以“风暴之主”AI 为例）：

```python
from .....StarkEngine.StarkServer import * # 导入 SE 服务端核心类
from .Actions import *                     # 导入自定义或内置的行为/条件节点

class StormLordAI(BTComponent):
    """风暴之主 AI 行为树组件"""
    
    def __init__(self, *args, **kwargs):
        # 必须调用父类构造函数
        BTComponent.__init__(self, *args, **kwargs)

    def BuildBehaviorTree(self):
        """
        行为树构建入口
        在此处通过组合节点、条件节点和行为节点搭建逻辑
        """
        
        # 构建逻辑树结构
        tree = Selector([
            Sequence([
                HasTarget(), # 检测目标
                Selector([
                    Sequence([
                        IsTargetInRange(0, 5), # 近距离技能
                        # ... 节点逻辑细节
                    ]),
                    Sequence([
                        MoveToTarget(speed=2.0) # 远程追击
                    ])
                ])
            ]),
            # 无目标时的逻辑
            RandomWaitAction(2, 3)
        ])

        # 【关键】将构建好的根节点返回
        return tree 
```

---

## 4. 开发要点提示
1.  **节点导入**: 确保所有使用的节点（如 `Selector`, `Sequence`, `HasTarget` 等）已通过 `import` 引入当前脚本。
2.  **构造函数**: 在 `__init__` 中务必执行 `BTComponent.__init__(self, *args, **kwargs)` 以确保引擎底层初始化正常。
3.  **调试建议**: 可以通过在节点中插入 `LogAction`（详见 AI 行为树基础）来在控制台实时观察行为树的决策流程。
