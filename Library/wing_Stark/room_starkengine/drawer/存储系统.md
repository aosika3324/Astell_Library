# [StarkEngine] 自动化存储系统 (存储系统.md)

> **AI 检索元数据**  
> **项目名称**: StarkEngine (wing_Stark)  
> **核心领域**: DataStorage, @AutoSave, 数据持久化, 存档重置, 跨文件数据共享  
> **主要目标**: 介绍如何通过装饰器实现数据的自动保存与加载，取代繁琐且难以维护的原生 ExtraData 接口。

---

## 1. 概述 (Overview)

为了解决网易原生 `SetExtraData` / `GetExtraData` 接口导致的数据管理混乱、效率低下等问题，StarkEngine 提供了一套**声明式**的数据持久化方案。

### 核心优势
*   **变动即保存**: 数据类中任何变量值的变动都会被自动检测并触发保存。
*   **重进即加载**: 存档重新进入时，会自动恢复上次保存的所有变量值。
*   **跨文件访问**: 像访问普通 Python 类属性一样读写存档数据，支持 IDE 补全。

---

## 2. 核心装饰器: `@DataStorage.AutoSave`

将一个普通的 Python 类声明为“自动存档类”。

### 2.1 参数说明
| 参数名 | 类型 | 描述 |
| :--- | :--- | :--- |
| **version** | `int` | **【重要】** 版本号。当代码中的版本与存档不符时，会丢弃旧数据并重置为代码默认值。 |
| **uid** | `str` | 存档唯一标识符。缺省时默认为 `module.ClassName`。 |

---

## 3. 开发实战

### 3.1 定义全局数据类
```python
from ...StarkEngine.StarkServer import DataStorage

# 声明该类为自动存档类，版本为 1
@DataStorage.AutoSave(version=1, uid="my_mod_global_data")
class GlobalData:
    day_count = 0        # 累计天数
    boss_defeated = False # Boss 击败状态
    player_scores = {}    # 玩家积分字典
```

### 3.2 数据的读写与共享
你可以在项目任何文件中通过 `import` 导入并直接操作该数据。

```python
# 文件：logic_system.py
from .config import GlobalData

class MySystem(BaseLevelComponent):
    def Start(self):
        # 1. 读取数据
        print(f"当前是第 {GlobalData.day_count} 天")
        
        # 2. 修改数据 (自动保存)
        GlobalData.day_count += 1
        GlobalData.boss_defeated = True
```

---

## 4. 运行逻辑与持久化表现

以 `day_count` 变量为例：

1.  **第一次进入存档**:
    *   读取代码默认值：`0`。
    *   逻辑执行 `+1` 变为 `1`。
    *   **结果**: 存档中保存 `1`。
2.  **退出游戏并重新进入**:
    *   引擎自动加载存档中的 `1`。
    *   逻辑执行 `+1` 变为 `2`。
    *   **结果**: 实现跨存档的数据延续。

---

## 5. 最佳实践
*   **结构变更**: 如果你大幅修改了数据类的变量结构（如删除了某个旧键并新增了复杂结构），建议提升 `version` 号以强制重置存档，避免数据错位导致的逻辑错误。
*   **集中管理**: 建议将所有的 `@AutoSave` 类集中定义在项目的 `config.py` 或 `storage_data.py` 中，方便维护。
